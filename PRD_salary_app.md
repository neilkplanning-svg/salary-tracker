## 1. מבוא

### 1.1 חזון
אפליקציה אישית לתיעוד נוכחות וחישוב שכר, המחליפה קובץ Excel קיים ומורכב. היא מאפשרת לעובד לתעד כניסה/יציאה יומית, לקבל **חישוב נטו משוער** מדויק, להזין את **תלוש השכר בפועל** כשהוא מגיע, להשוות משוער מול בפועל, לנהל היסטוריה רב‑שנתית עם גרפים, ולייצא/לייבא ל‑Excel. אותו קוד רץ על **מובייל** (אנדרואיד + iOS) וכ**אפליקציית WEB** ל‑PC, עם סנכרון מבוסס‑קובץ דרך **OneDrive**.

### 1.2 פלטפורמות וקהל יעד
- **פלטפורמות:** אנדרואיד, iOS, ו‑WEB ל‑PC — מבסיס קוד **Flutter** יחיד.
- **קהל:** המחבר וחבריו לעבודה ב‑IAI / MALAT, החולקים אותו מבנה שכר. כל משתמש מריץ מופע עצמאי עם נתונים מקומיים משלו.
- **מודל השכר:** מותאם למבנה של IAI (שכר יסוד + כוננות, כללי שעות נוספות, קרן עזרה, ניכויים אופייניים), אך כל **הפרמטרים האישיים** ניתנים לעריכה לכל משתמש.

### 1.3 מטרות (Goals)
1. לבטל חישובי שכר ידניים ב‑Excel ולהפוך אותם לתהליך מהיר ונייד.
2. לספק הערכת נטו מדויקת **ברמת דיוק האקסל הקיים** עוד לפני קבלת התלוש.
3. לאפשר תיעוד תלוש בפועל והשוואה מיידית מול המשוער.
4. לשמר היסטוריה רב‑שנתית רציפה עם גרפים, ללא תלות בקובץ Excel יחיד ושביר.
5. לאפשר עבודה חלקה בין מובייל ל‑PC דרך OneDrive — **באפס עלות חוזרת וללא שרת**.

### 1.4 מדדי הצלחה (Success Metrics)
- **דיוק:** הפער בין הנטו המשוער של האפליקציה לבין הנטו בתלוש בפועל קטן מ‑1% (לאחר הזנת פרמטרים תקינים). אימות מול האקסל המקורי: סטייה ≤ ₪1 לאותם קלטים.
- **אימוץ:** המחבר מפסיק להשתמש באקסל לחישוב החודשי השוטף; לפחות 3 חברים לעבודה משתמשים במופע עצמאי.
- **רציפות נתונים:** ייצוא→ייבוא JSON וכן ייצוא→ייבוא Excel הם round‑trip ללא אובדן נתונים.
- **שטף סנכרון:** מחזור "עריכה במובייל → שמירה ל‑OneDrive → עריכה ב‑WEB → משיכה חזרה למובייל" עובד בלי התערבות ידנית מעבר לבחירת הקובץ.

## 2. הגדרת הבעיה
החישוב מתבצע כיום בקובץ Excel גדול הכולל גיליון מחשבון חודשי, גיליון סטטוס רב‑שנתי, וגיליון לכל שנה (2017 ואילך), בתוספת מודולי "הפחתות שכר זמני" ו"קרן עזרה". הבעיות:

- **ידני ושביר:** נוסחאות מורכבות בקובץ יחיד, רגיש לטעויות ולמחיקה בשוגג; אין גיבוי מסודר.
- **לא נייד:** אי אפשר "ללחוץ כניסה/יציאה" בזמן אמת מהטלפון; ההזנה דורשת מחשב.
- **חיכוך בתחזוקה:** עדכון מדרגות מס, נקודות זיכוי וערכי מדינה דורש עריכת נוסחאות ידנית בתאים.
- **אין השוואה נוחה משוער↔בפועל:** הצלבת התלוש מול ההערכה נעשית ידנית.
- **קשה לשיתוף:** קולגות שרוצים כלי דומה נדרשים להעתיק ולהבין קובץ אישי ומותאם.

**עלות אי‑פתרון:** המשך עבודה איטית ומועדת לטעויות, סיכון לאובדן היסטוריה רב‑שנתית, וחוסר נגישות מהנייד.

## 3. סקירת הפתרון / תכונות

### 3.1 ארכיטקטורה ברמת‑על
- בסיס קוד **Flutter יחיד** → אנדרואיד, iOS ו‑WEB (build סטטי הנפתח בקליק).
- **אחסון מקומי מבוסס‑JSON** במכשיר/בדפדפן; אותו מבנה JSON הוא גם פורמט הסנכרון.
- **OneDrive כמתווך מבוסס‑קובץ:** ה‑WEB פותח/שומר קובץ JSON מתוך תיקיית OneDrive המסונכרנת ב‑PC (File System Access API); המובייל מייצא/מייבא את אותו קובץ ל‑OneDrive דרך מסך השיתוף של מערכת ההפעלה. **ללא שרת.**
- **מנוע חישוב יחיד** המשותף לכל היעדים, מאומת מול האקסל המקורי.

### 3.2 מודולים ותכונות
1. **שעון נוכחות** — כפתור כניסה/יציאה בזמן אמת (מתייג זמן נוכחי) **וגם** עריכה ידנית של שעת התחלה/סיום. רשת ימי החודש כוללת: שעות רגילות, "שעות אפס", שעות נוספות, יום השתלמות, סימון נוכחות/היעדרות.
2. **מנוע חישוב שכר משוער** — חישוב מלא: שעות → שעות נוספות → ברוטו → מדרגות מס, ביטוח לאומי, מס בריאות, פנסיה, קרן השתלמות, נקודות זיכוי, זקיפות → **נטו משוער** (ראו §5.3).
3. **תלוש בפועל** — לאחר קבלת התלוש מזינים: ברוטו, נטו, שעות נוספות שאושרו, תוספות/מענקים שניתנו, והערות חופשיות; האפליקציה מציגה **השוואה משוער↔בפועל**.
4. **מודול "הפחתות שכר זמני"** — הפחתות חודשיות משכר רגיל ומשעות נוספות, מענק רבעוני וניכוי מענק.
5. **מודול "קרן עזרה"** — הפקדות, יתרת חיסכון, הלוואות, בקשת הלוואה והחזרים.
6. **היסטוריה ודשבורד** — תצוגה חודשית ושנתית; גרפים (ראו §3.3); נתונים רב‑שנתיים (2017 ואילך) הנשמרים כ‑**snapshot מחושב לכל חודש**.
7. **הגדרות / פרמטרים** — עריכת כל ערכי החישוב (לאומיים ואישיים).
8. **ייצוא/ייבוא** — Excel (קריאת/כתיבת נתוני שכר) ו‑JSON (סנכרון/גיבוי).
9. **סנכרון וגיבוי** — קובץ JSON ב‑OneDrive (ראו §5.4).

### 3.3 חלוקת היקף — MoSCoW

**Must‑Have (P0, v1)**
- שעון נוכחות (כפתור + עריכה ידנית) ורשת חודשית.
- מנוע חישוב נטו משוער מלא ומדויק.
- תלוש בפועל + השוואה משוער↔בפועל.
- מודולי "הפחתות שכר זמני" ו"קרן עזרה".
- היסטוריה חודשית/שנתית כ‑snapshots.
- 3 גרפים: ברוטו/נטו שנתי לאורך השנים; ממוצע חודשי לאורך השנים; אחוז שינוי הכנסה מול אינפלציה.
- מסך הגדרות פרמטרים מלא.
- ייצוא/ייבוא Excel + ייצוא/ייבוא JSON.
- גרסה ריקה + טעינת נתונים (§5.7).
- ממשק עברית RTL, מטבע ₪, מצב כהה/בהיר.
- WEB סטטי הנפתח בקליק + טעינה/שמירה של קובץ OneDrive (Edge/Chrome).

**Should‑Have (P1, fast‑follow)**
- גרף "מגמת ברוטו/נטו חודשי בתוך השנה".
- מדדים משניים בדשבורד: יחס נטו/ברוטו, אחוז משרה ממוצע, סך שעות ללא תמורה.
- התראות/תזכורות (למשל "הזן תלוש החודש").
- fallback להעלאה/הורדה ידנית בדפדפנים ללא File System Access API.

**Future (P2 — לא נבנה כעת, אך הארכיטקטורה תתמוך)**
- סנכרון אוטומטי מלא מול OneDrive דרך **Microsoft Graph** (OAuth/MSAL, רישום Azure).
- מערכת פרמטרים **תלוית‑שנה** (אם יידרש חישוב היסטורי מחדש).
- סוגי הכנסה/רכיבים נוספים מעבר למבנה הנוכחי.

### 3.4 מחוץ להיקף (Non‑Goals)
- **אין מסד נתונים אוטומטי של חוקי מס.** המשתמש מתחזק את ערכי המדינה ידנית (עיקרון זהה לזה שהוביל אצל המחבר להימנעות ממסד תרופות אוטומטי בפרויקט בריאות — הימנעות מאחריות על נתונים רגולטוריים משתנים).
- **אין שרת אחורי ואין חשבונות משתמש.** הנתונים מקומיים; OneDrive הוא גיבוי/סנכרון בלבד.
- **אין סנכרון בזמן אמת בין מכשירים** ב‑v1 (קובץ משותף בלבד).
- **אין ייעוץ מס/משפטי.** האפליקציה היא כלי הערכה אישי, לא תחליף לתלוש הרשמי.
- **אין שיתוף נתונים בין משתמשים שונים.** כל מופע עצמאי.

## 5. דרישות טכניות

### 5.1 Stack
- **Framework:** Flutter (Dart), יעדי build: `apk`/`appbundle` (אנדרואיד), iOS, ו‑`web` (build סטטי).
- **State management:** Riverpod.
- **אחסון מקומי:** מודל נתונים מבוסס‑JSON שמור מקומית (למשל Hive או קובץ JSON יחיד דרך `path_provider`; בדפדפן — IndexedDB/אחסון מקומי דרך אותה שכבת abstraction). **המקור הקנוני הוא מסמך JSON אחד** התואם לפורמט הייצוא/סנכרון.
- **Excel:** קריאה/כתיבה דרך חבילת Dart ל‑`.xlsx` (למשל `excel` או `syncfusion_flutter_xlsio`).
- **גרפים:** `fl_chart`.
- **לוקליזציה:** `intl` עם `he_IL`, RTL, מטבע ₪, פורמט תאריך עברי/לועזי.
- **WEB ↔ קובץ:** File System Access API דרך JS interop (`dart:js_interop` או חבילה מתאימה), עם fallback להורדה/העלאה רגילה.
- אין תלות בשירות צד‑שלישי בתשלום; אין רכיב שרת.

### 5.3 מנוע החישוב — pipeline ופרמטרים
החישוב חייב לשחזר את לוגיקת האקסל המקורי **במדויק**. כל המספרים הם פרמטרים (אין hard‑coding של ערכים אישיים). הזרימה החודשית:

1. **קלט שעות:** מתוך רשת הימים — שעות רגילות, שעות אפס, שעות נוספות, ימי השתלמות, היעדרות, ואחוז משרה / שעות משרה.
2. **רכיבי ברוטו:** `baseSalary` (לפי נוסחת `baseConst + perHourConst × positionFactor`, מוכפל ב‑`adjustFactor`) + כוננות + שכר עידוד + רכב + טלפון + תוספות.
3. **שעות נוספות:** חישוב רב‑שכבתי לפי ערך שעה נוספת ומדרגות (125%/150%), תוספת שבת/חג ותוספת כוננות, מול מכסת שעות נוספות ושעות שאושרו. **יש לשחזר את נוסחת השעות הנוספות מהאקסל בדיוק** (כולל לוגיקת המכסה והמדרגות) ולאמת מול ערכי הקובץ.
4. **זקיפות:** סכימת `imputations` והוספתן לבסיס החייב במס.
5. **מס הכנסה (לפי מדרגות):** עבור כל מדרגה `i` עם `[min, max, rate]`:
   `tax_i = clamp(taxable − min_i, 0, max_i − min_i) × rate_i`
   `incomeTax = Σ tax_i − (creditPointsQty × creditPointValue)`, **לא קטן מ‑0**.
6. **ביטוח לאומי ומס בריאות:** חישוב מדורג זהה במבנה (מדרגה מופחתת עד סף, מדרגה מלאה עד תקרה).
7. **פנסיה:** `pensionRateEmployee × בסיס פנסיוני`. **קרן השתלמות:** `trainingFundRateEmployee × min(בסיס, trainingFundCap)`.
8. **ניכויים נוספים:** סכימת `otherDeductions`.
9. **נטו משוער:** `gross − (incomeTax + nationalInsurance + healthTax + pension + trainingFund + otherDeductions)`.
10. **נטו לאחר הפחתות:** הפחתת רכיבי "הפחתות שכר זמני" והחזרי "קרן עזרה" → `netAfterReductions`.

**ערכי ברירת מחדל:** פרמטרים **לאומיים** (מדרגות מס, ערך נקודת זיכוי, ב"ל, מס בריאות, תקרת קרן השתלמות) ייטענו עם הערכים הרשמיים הנוכחיים בישראל כברירת מחדל הניתנת לעריכה — מבנה מדרגות מס שולי של 10%/14%/20%/31%/35%/47%/50%, ספים לפי הפרסום הרשמי העדכני. פרמטרים **אישיים** וכל נתוני השכר/השעות מתחילים **ריקים**.

### 5.4 סנכרון וגיבוי (OneDrive)
- **WEB (אופציה שנבחרה):** טעינה/שמירה של קובץ `salary-data.json` מתוך **תיקיית OneDrive המסונכרנת ב‑PC** דרך File System Access API (Edge/Chrome). שמירה כותבת חזרה לאותו file handle; סנכרון OneDrive עצמו מפיץ את העדכון.
- **מובייל:** ייצוא קובץ ה‑JSON ושמירתו ל‑OneDrive דרך מסך השיתוף של מערכת ההפעלה; ייבוא דרך בורר קבצים.
- **זרימת bidirectional:** מובייל שומר ל‑OneDrive → WEB פותח, עורך, שומר חזרה → מובייל מייבא מחדש.
- **זיהוי גרסה:** השוואת `appMeta.lastModified`; **"הכתיבה האחרונה מנצחת"** + **התראה** אם הקובץ שנטען חדש מזה שבמכשיר (מניעת דריסה לא‑מכוונת).
- **ללא שרת, ללא חשבונות, אפס עלות.**

### 5.5 ייצוא/ייבוא Excel
- **ייצוא:** קובץ `.xlsx` עם גיליונות מקבילים למבנה הקיים (חודשי, סטטוס/שנתי, היסטוריה).
- **ייבוא:** קריאת נתוני שכר/שעות מ‑Excel למבנה ה‑JSON, לטעינת הנתונים האישיים לאחר שהאפליקציה עובדת (§5.7).
- ה‑Excel הוא ערוץ עבודה אנושי; **JSON** הוא פורמט הסנכרון/הגיבוי.

### 5.6 אירוח ו"פתיחה בקליק"
- ה‑WEB מפורסם כ‑**build סטטי** (למשל GitHub Pages) ונפתח דרך URL בלבד — **ללא התקנה, ללא localhost, ללא שרת**.
- אין צורך בתהליך בנייה אצל המשתמש; הוא פותח קישור ומתחיל לעבוד.

### 5.7 גרסה ריקה + טעינת נתונים
- ברירת מחדל: **אפליקציה ריקה** מנתונים אישיים (פרמטרים לאומיים מאוכלסים מראש, ניתנים לעריכה).
- טעינת נתונים אישיים דרך **ייבוא JSON** או **ייבוא Excel**.

### 5.8 עיצוב, RTL ונגישות
- ממשק **עברית RTL** מקצה לקצה; מטבע ₪; פורמטי תאריך/שעה ישראליים.
- **מצב כהה ובהיר** (`system|light|dark`).
- ערכת נושא ניתנת להתאמה; ברירת מחדל בהשראת מותג אישי (navy `#14274E`, gold `#C9A24B`).
- טפסים נוחים למובייל; כפתור כניסה/יציאה בולט ונגיש במסך הראשי.

### 5.9 התאמה ל‑`CLAUDE.md`
מצורף קובץ `CLAUDE.md` המגדיר stack, מבנה תיקיות, מוסכמות קוד, מודל הנתונים ומפרט מנוע החישוב. כל הדרישות כאן מיושרות אליו, והוא מקור האמת לכללי הארכיטקטורה והקוד.

## 7. אילוצים
- **אפס עלות חוזרת** מועדפת; ללא שרת וללא שירות בתשלום.
- **פרטיות / local‑first:** נתונים נשמרים מקומית; OneDrive הוא גיבוי/סנכרון בלבד; אין חשבונות משתמש.
- **מפתח יחיד** עם Claude Code; אין צוות; **ללא דדליין קשיח**.
- **דפדפן:** חוויית הקובץ החלקה ב‑WEB נתמכת ב‑Edge/Chrome (File System Access API); דפדפנים אחרים — fallback להעלאה/הורדה (P1).
- **הפצה:** תוחלט בהמשך. שים לב: פרסום ב‑App Store דורש חשבון מפתח של Apple (~99$ לשנה) ומתנגש עם יעד אפס‑עלות; אנדרואיד מאפשר הפצת APK ישירה ללא עלות חוזרת.
- **מודל שכר ספציפי ל‑IAI/MALAT**; לא כללי לכל מעסיק.
- **ללא ייעוץ מס/משפטי** וללא תחליף לתלוש הרשמי.
- **שפה:** עברית RTL; מונחים טכניים באנגלית.

## 8. מקרי קצה והערות

### 8.1 מקרי קצה
- **שכחת החתמה:** יום עם "כניסה" ללא "יציאה" → התראה והשלמה ידנית.
- **חודש חלקי / תחילת עבודה באמצע חודש** → חישוב יחסי לפי הימים שהוזנו.
- **קונפליקט עריכה** מובייל↔WEB על אותו קובץ → "הכתיבה האחרונה מנצחת" + התראת גרסה.
- **מעבר שנה** → יצירת `yearSummary` חדש; שמירת רציפות היסטורית.
- **ערכים אפס/שליליים** (שעות, סכומים) → ולידציה ומניעת תוצאות לא תקינות.
- **שעות נוספות חריגות / מעבר למכסה** → לפי לוגיקת המכסה מהאקסל.
- **שינוי פרמטר באמצע שנה** → משפיע רק על חישובים חדשים; snapshots קיימים נשמרים.
- **קובץ OneDrive חסר/לא מסונכרן** → הודעת שגיאה ברורה ואפשרות לבחור קובץ אחר.
- **דפדפן ללא File System Access API** → fallback להעלאה/הורדה ידנית.
- **פורמט Excel לא תואם בייבוא** → הודעת שגיאה והנחיה למבנה הנדרש.
- **אזור זמן ישראל** ושעון קיץ → תיוג זמנים עקבי.
- **שחזור מגיבוי** פגום/חלקי → ולידציית `schemaVersion` והודעה.

### 8.2 הערות ושכבות עתידיות (P2)
- **סנכרון אוטומטי מלא** מול OneDrive דרך Microsoft Graph (MSAL, רישום Azure) — תלוי במדיניות ה‑IT הארגוני.
- **פרמטרים תלויי‑שנה** — אם יידרש בעתיד חישוב היסטורי מחדש (כרגע: מערכת ערכים נוכחית אחת + snapshots).
- **מדדים/גרפים נוספים** ורכיבי הכנסה נוספים.

### 8.3 אימות מול האקסל המקורי
- יש להקים **מערך בדיקות** המזין סדרת קלטים מייצגת ומשווה את פלט המנוע (ברוטו, מס, ב"ל, מס בריאות, פנסיה, השתלמות, שעות נוספות, נטו) מול ערכי האקסל, עם סטייה מותרת ≤ ₪1.
- **נוסחת השעות הנוספות** הלא‑סטנדרטית שבאקסל תילקח כמקור אמת ותשוחזר ותאומת תא‑מול‑תא במהלך הפיתוח.
